# 🚀 Auto GitHub Push CLI

Умный Bash-скрипт для автоматической и интерактивной выгрузки Bash-скриптов (`.sh`) в ваши публичные репозитории GitHub прямо из файлового менеджера или терминала Linux.

Скрипт автоматически сканирует рабочую директорию (включая все подпапки), сверяет файлы с репозиторием по хэш-сумме и размеру (защита от повторной отправки дубликатов), запрашивает целевой репозиторий через графический интерфейс и дает удобную паузу для редактирования документации (`README.md`) перед отправкой.

---

## ✨ Основные возможности

* **Интерактивный выбор репозитория**: Динамически загружает список всех ваших публичных репозиториев через официальный GitHub API.
* **Глубокий рекурсивный поиск**: Находит все `.sh` файлы во всех вложенных папках.
* **Интеллектуальное сравнение (Защита от дубликатов)**: Сравнивает не просто имена, а реальное содержимое файлов (`git diff` по хэш-суммам). Если файл не изменился, скрипт не будет тратить время на его отправку.
* **Безопасность**: Скрипт никогда не отправит сам себя на GitHub, даже если его копия лежит во вложенной папке с исходниками.
* **Удобная пауза для README**: Перед финальным коммитом и пушем скрипт делает паузу и выводит окно-напоминание, чтобы вы успели обновить описание репозитория.
* **Графический интерфейс**: Полная интеграция с KDE Plasma благодаря использованию `kdialog`.

---

## 📋 Системные требования

Скрипт разработан и протестирован на **Fedora Linux** (с окружением KDE Plasma), но будет работать на любом дистрибутиве при наличии следующих пакетов:
1. `git` (управление версиями)
2. `kdialog` (для отрисовки графических окон)
3. `gh` (официальный GitHub CLI)

## 🛠️ Инсталляция и настройка (Шаг за шагом)

### Шаг 1: Установка зависимостей

Откройте терминал и выполните команду для установки необходимых утилит:

```bash
# Для Fedora Linux
sudo dnf install git gh kdialog -y

# Для Ubuntu / Debian (потребуется kdialog из пакета kde-cli-tools)
# sudo apt update && sudo apt install git gh kde-cli-tools -y
```

### Шаг 2: Авторизация в GitHub CLI

Чтобы скрипт мог видеть ваши репозитории и отправлять в них код, необходимо один раз войти в свой аккаунт через консольный клиент GitHub:

1. Запустите команду авторизации:
   ```bash
   gh auth login
   ```
2. Ответьте на вопросы в терминале:
   * **Where do you use GitHub?** -> Выберите `GitHub.com` и нажмите *Enter*.
   * **What is your preferred protocol for Git operations?** -> Выберите `HTTPS` и нажмите *Enter*.
   * **Authenticate Git with your GitHub credentials?** -> Нажмите **`Y`** (Yes) и затем *Enter*.
   * **How would you like to authenticate GitHub CLI?** -> Выберите `Login with a web browser` и нажмите *Enter*.
3. Терминал покажет **8-значный код** (например, `2FF5-0A75`). Скопируйте его.
4. Нажмите *Enter*. В браузере откроется страница авторизации GitHub.
5. Войдите в свой аккаунт GitHub, вставьте скопированный код и подтвердите авторизацию (`Authorize github`).
6. Вернитесь в терминал — там появится сообщение об успешном входе: `Logged in as USERNAME`.

## 🚀 Как использовать скрипт

1. Скачайте файл скрипта `Auto_Push_Github.sh` и поместите его в любую папку на вашем компьютере.
2. Сделайте скрипт исполняемым. Это нужно сделать только один раз:
   ```bash
   chmod +x Auto_Push_Github.sh
   ```
3. Перенесите или скопируйте любые `.sh` файлы (или папки со скриптами) в ту же директорию, где лежит `Auto_Push_Github.sh`.
4. Запустите скрипт двойным кликом в файловом менеджере (выберите "Запустить в терминале") или через терминал:
   ```bash
   ./Auto_Push_Github.sh
   ```

### Что произойдет дальше:
1. Появится окно со списком ваших репозиториев GitHub — выберите нужный кликом мыши.
2. Скрипт автоматически сравнит файлы с версией на GitHub. Если будут найдены идентичные файлы, он покажет их список и пропустит их.
3. Перед отправкой появится окно-пауза: *"Сейчас самое время отредактировать README.md!"*.
4. Спокойно отредактируйте описание проекта в `README.md` во временной папке (путь к ней будет написан в окне).
5. Нажмите **Да** в диалоговом окне — скрипт выполнит автоматический коммит, синхронизирует ветки и выгрузит изменения на GitHub!
